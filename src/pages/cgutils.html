Config Files

Config for file downloads and stacked commands are saved as json files. Stacked commands are saved at ./config/cgutils/stacks and downloads are saved at ./config/cgutils/downloads



File Downloads

File downloads can only be initiated by a server admin. The URL can be specified in config or typed ingame. The JSON format is shown below.

{
 "name":"unique_name_with_no_spaces",
 "saveLocation":"save directory relative to server jar",
 "url":"http://example.com/myfile.txt",
 "group":"group_name"
}

There are 3 commands that can be used to download files in CGUtils. These are:

  • - /update get <unique_name_with_no_spaces>: This downloads the file with the unique name
  • - /update group <group_name>: This downloads all files in the specified group
  • - /update url <url> <location>: Downloads the file found at url to location

Example file download JSON file:

{
 "name":"tinkers",
 "saveLocation":"scripts/",
 "url":"http://inky2013.github.io/mt_scripts/tinkers.zs",
 "group":"minetweaker"
}

We could then download "tinkers.zs" using any one of the following commands:

  • - /update get tinkers
  • - /update group minetweaker
  • - /update url http://inky2013.github.io/mt_scripts/tinkers.zs scripts/


Stacked Commands

Stacked commands are a list of commands that all run in sequence when the command is run. A simple example is shown below which will summon a cow, pig, sheep and chicken at the sender.

{
    "name":"summonfood",
    "requiresop":false,
    "commands": [
        "execute {{sender}} ~ ~ ~ summon Cow ~ ~ ~",
        "execute {{sender}} ~ ~ ~ summon Sheep ~ ~ ~",
        "execute {{sender}} ~ ~ ~ summon Pig ~ ~ ~",
        "execute {{sender}} ~ ~ ~ summon Chicken ~ ~ ~"
    ]
}

Arguments can also be added to the command.

{
    "name":"subtitle",
    "requiresop":true,
    "commands": [
        "title {{0}} title {\"text\":\"\"}",
        "title {{0}} subtitle {\"text\":\"{{1}}}\"}"
    ]
}

The command will then be run with /subtitle <player> <text>



Sudo Command

The sudo command has 2 main uses:

  • - To run an admin command from inside a stacked command
  • - To run an admin command from a fake player (eg: open computers)

It has very simple usage: /sudo <command with spaces>
